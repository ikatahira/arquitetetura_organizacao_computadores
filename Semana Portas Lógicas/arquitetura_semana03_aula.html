LOGIC: Portas Lógicas e Álgebra Booleana

Portas lógicas são os átomos da computação digital. Tudo que um processador faz — somar, comparar, decidir — se reduz a combinações de AND, OR e NOT. Entender como elas funcionam é a diferença entre um desenvolvedor que usa hardware e um que entende hardware. Nesta missão você vai simular, construir e analisar circuitos reais.

// 3.1 — O que são portas lógicas?

George Boole formalizou em 1854 a álgebra que opera apenas com dois valores: verdadeiro e falso (1 e 0). Claude Shannon mostrou em 1937 que essa álgebra podia ser implementada com circuitos elétricos. O resultado: toda operação computacional se reduz a combinações de três operações fundamentais — AND, OR, NOT.

Fisicamente, uma porta lógica é um conjunto de transistores conectados de forma específica. Uma porta AND com dois transistores em série: ambos precisam conduzir para a saída ter tensão. Uma porta OR com transistores em paralelo: qualquer um conduzindo já aciona a saída.

No Arduino e FPGA: Quando você escreve if (a && b) em código, o compilador gera instruções que eventualmente acionam portas AND no processador. Em FPGAs, você pode desenhar as próprias portas lógicas explicitamente em VHDL ou Verilog.

// 3.4 — Álgebra Booleana: Leis Fundamentais

A álgebra booleana permite simplificar expressões lógicas, reduzindo o número de portas em um circuito — menos portas = menor consumo de energia, menor área no chip, maior velocidade.

A · 0 = 0    A · 1 = A
// Lei do Elemento Nulo / Identidade AND
A + 0 = A    A + 1 = 1
// Lei do Elemento Nulo / Identidade OR
A · A = A    A + A = A
// Lei da Idempotência
A · Ā = 0    A + Ā = 1
// Lei do Complemento
NOT(NOT A) = A
// Lei da Dupla Negação
A·B = B·A    A+B = B+A
// Lei Comutativa
A(B+C) = AB + AC
// Lei Distributiva AND sobre OR
A+(B·C) = (A+B)·(A+C)
// Lei Distributiva OR sobre AND
NOT(A·B) = Ā + B̄
// Teorema De Morgan (1)
// NAND = OR das inversões
NOT(A+B) = Ā · B̄
// Teorema De Morgan (2)
// NOR = AND das inversões
A + A·B = A
// Lei da Absorção
// A já contém A·B
A · (A+B) = A
// Lei da Absorção (2)
De Morgan na prática: NOT(A AND B) = (NOT A) OR (NOT B). Em código: !(a && b) é equivalente a (!a || !b). Você usa isso toda vez que inverte uma condição composta em if-else!

// 3.6 — Exemplo de Simplificação: Do Problema ao Circuito Mínimo

Problema: Um sistema de alarme dispara (S=1) quando: (a) é noite E a janela está aberta, OU (b) é noite E a porta está aberta, OU (c) a senha está errada.

Variáveis: N=noite, J=janela aberta, P=porta aberta, E=senha errada.

Expressão inicial: S = (N·J) + (N·P) + E

Simplificação:

S = N·J + N·P + E

S = N·(J + P) + E ← fatoramos N usando distributividade

Resultado: em vez de 3 portas AND + 2 portas OR = 5 portas, usamos apenas 1 AND + 2 OR = 3 portas. Um chip com essa lógica seria 40% menor.

No Arduino: Se você programa if((night && window) || (night && door) || wrongPwd), o compilador GCC já faz essa simplificação automaticamente em nível de otimização O2+. Mas para FPGAs e hardware embarcado com recursos limitados, você precisa fazer manualmente.

SIMULAÇÃO: Explorando Portas no CircuitVerse
⏱ 30min ★ 100 XP CircuitVerse · circuitverse.org
Use o CircuitVerse (simulador online gratuito, sem instalação) para montar e testar circuitos com as 6 portas lógicas básicas. O CircuitVerse foi desenvolvido especificamente para circuitos digitais lógicos — é mais direto para este fim do que simuladores gerais.
// PASSO A PASSO:
  1. 01
    Acesse circuitverse.org/simulator. Clique em "Launch Simulator" — não precisa de conta para simular, mas crie uma conta gratuita para salvar seu projeto. Nome do projeto: portas_logicas_seu_nome
  2. 02
    No painel esquerdo, expanda a categoria Gates. Localize AND, OR, NOT, NAND, NOR e XOR. Arraste um de cada para o canvas. Organize-os em fileiras para ficar legível.
  3. 03
    Para cada porta: na categoria Input, arraste elementos Input (botão toggle) para conectar às entradas da porta. Na categoria Output, arraste um Output (LED) para a saída. Conecte clicando nos terminais — um fio verde aparece ao conectar corretamente.
  4. 04
    A simulação roda em tempo real no CircuitVerse — não há botão "iniciar". Clique nos inputs para alternar entre 0 e 1 e observe o LED de saída mudar imediatamente. Teste TODAS as combinações de entrada: 4 combinações para portas de 2 entradas, 2 para NOT.
  5. 05
    Para cada porta testada, preencha a tabela verdade manualmente no caderno ou num Google Docs. Use o atalho Ctrl+P no CircuitVerse para imprimir/exportar como imagem, ou tire screenshot. Compile em simulacao_portas_seu_nome com as tabelas e evidências visuais.
  6. 06
    Bônus: monte um circuito combinacional no mesmo projeto que implemente a expressão: S = (A·B) + (NOT C) — use 3 inputs, 1 AND, 1 NOT, 1 OR e 1 output. Verifique todas as 8 combinações com o gerador de tabela verdade desta aula.
No CircuitVerse, se o fio aparecer vermelho ao conectar, há conflito de sinal — desfaça e tente novamente. Para excluir um componente: selecione e pressione Delete. Para salvar online: Menu > Save (requer login). Tutorial rápido: busque "circuitverse simulator tutorial" no YouTube — há vídeos em PT-BR.

⏱ 35min ★ 100 XP Papel + Google Docs
Simplifique as expressões booleanas abaixo aplicando as leis da álgebra booleana passo a passo. Para cada passo, indique qual lei foi aplicada. Verifique usando o gerador de tabela verdade desta aula.
// EXPRESSÕES PARA SIMPLIFICAR:
  1. A
    S = A·B + A·B̄ — Dica: fatore A. O resultado é surpreendentemente simples.
  2. B
    S = (A + B) · (A + B̄) — Aplique distributividade e lei do complemento.
  3. C
    S = A·B·C + A·B·C̄ + A·B̄·C — Fatore em etapas. Resposta tem 2 termos.
  4. D
    S = NOT(NOT A · NOT B) — Aplique De Morgan. Que porta isso implementa?
  5. E
    S = A + A·B + A·B·C — Lei da absorção aplicada duas vezes.
  6. F
    Para cada expressão simplificada: (1) mostre todos os passos com a lei aplicada entre parênteses, (2) verifique usando o gerador de tabela verdade, (3) indique quantas portas a expressão original precisaria vs a simplificada.
Para a expressão D: NOT(NOT A · NOT B). De Morgan diz NOT(X·Y) = NOT(X) + NOT(Y). Aplique com X=NOT A e Y=NOT B. Simplifique a dupla negação. Reconhece o resultado?

PROJETO: Sistema de Alarme com Lógica Combinacional
⏱ 25min ★ 100 XP draw.io + Google Docs
A missão mais importante da semana: projetar um circuito lógico completo a partir de uma especificação funcional real, seguindo o processo profissional de design digital.
// ESPECIFICAÇÃO DO SISTEMA:
  1. SPEC
    Um servidor de data center deve disparar um alerta (S=1) nas seguintes condições: (1) Temperatura ALTA (T=1) E o sistema de resfriamento DESLIGADO (R=0), OU (2) Temperatura ALTA (T=1) E Umidade ALTA (U=1), OU (3) Tensão de alimentação FORA DO RANGE (V=1). Projetar o circuito mínimo.
  2. 01
    Formular a expressão: A partir da especificação, escreva a expressão booleana literal. Identifique as variáveis (T, R, U, V) e as operações.
  3. 02
    Construir a tabela verdade: Com 4 variáveis, há 2⁴ = 16 linhas. Preencha manualmente para todos os casos. Verifique com o gerador desta aula.
  4. 03
    Simplificar a expressão: Aplique as leis da álgebra booleana para reduzir o número de portas. Dica: lembre-se do exemplo do alarme na seção 3.6 — a mesma técnica se aplica aqui.
  5. 04
    Desenhar o circuito: No draw.io (app.diagrams.net), desenhe o circuito completo usando os símbolos corretos das portas lógicas. As entradas devem entrar pela esquerda, a saída sair pela direita.
  6. 05
    Documentar: Compile num Google Docs: expressão original → simplificação passo a passo → tabela verdade → imagem do circuito → análise: quantas portas economizou?
No draw.io: pesquise na barra de formas "AND gate" — existem formas padronizadas para portas lógicas (padrão IEEE/ANSI). Para 4 variáveis, use "Shapes > Electrical > Logic Gates".
