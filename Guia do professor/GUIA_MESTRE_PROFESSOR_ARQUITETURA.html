🎯 Introdução e Instruções de Uso

Sobre Este Guia

Este guia mestre foi desenvolvido para auxiliar professores na aplicação e correção das atividades da disciplina de Arquitetura e Organização de Computadores. Cada exercício contém:

  • Resolução completa e detalhada
  • Critérios objetivos de correção
  • Rubricas de avaliação com pontuação
  • Códigos comentados (quando aplicável)
  • Diagramas e esquemas de montagem
  • Variações de respostas aceitáveis
  • Erros comuns dos alunos
  • Dicas pedagógicas
Como Usar Este Guia
  1. Preparação da Aula: Leia a resolução completa antes da aula para antecipar dúvidas
  2. Durante a Aula: Use as dicas pedagógicas para orientar os alunos
  3. Correção: Utilize as rubricas e critérios objetivos para avaliar
  4. Feedback: Identifique erros comuns e forneça feedback direcionado
💡 Dica Importante: Este guia não deve ser compartilhado com os alunos antes da realização das atividades. Mantenha-o como material de apoio docente.

Exercício 1: Pesquisa sobre Gerações de Computadores

Objetivo: Investigar e apresentar a evolução tecnológica dos computadores

Duração: 30 minutos | Peso: 10 pontos

📋 Descrição da Atividade

Os alunos devem pesquisar e criar uma tabela comparativa das 5 gerações de computadores, identificando:

  • Período temporal de cada geração
  • Tecnologia principal utilizada (válvulas, transistores, CI, etc.)
  • Exemplos de computadores marcantes
  • Características principais (velocidade, tamanho, custo, consumo de energia)
  • Principais avanços introduzidos

Geração Período Tecnologia Exemplos Características
1ª Geração 1940-1956 Válvulas eletrônicas (tubos de vácuo) ENIAC, UNIVAC, EDVAC • Muito grandes e pesados
• Alto consumo de energia (até 150kW)

2ª Geração 1956-1963 Transistores IBM 7094, CDC 1604, UNIVAC 1107 • Menores e mais confiáveis

• Linguagens: Assembly, FORTRAN, COBOL

3ª Geração 1964-1971 Circuitos Integrados (CI) IBM System/360, PDP-8, CDC 6600 • Miniaturização significativa
• Multiprogramação e sistemas operacionais
• Operações: ~1.000.000/segundo
• Menor custo de produção
• Maior confiabilidade
4ª Geração 1971-presente Microprocessadores (LSI/VLSI) Intel 4004, IBM PC, Apple II, Macintosh • Computadores pessoais (PCs)
• Milhões de transistores em chip único
• Operações: bilhões/segundo (GHz)
• Interface gráfica (GUI)
• Redes e Internet
5ª Geração Presente-Futuro IA, Computação Quântica, Nanotecnologia IBM Watson, D-Wave, computadores quânticos • Inteligência Artificial e Machine Learning
• Processamento paralelo massivo
• Computação quântica (qubits)
• Reconhecimento de voz e imagem
• Computação cognitiva

Exercício 2: Identificação de Componentes

Objetivo: Reconhecer componentes em um computador real

Duração: 35 minutos | Peso: 10 pontos

📋 Descrição da Atividade

Os alunos devem identificar e descrever a função dos principais componentes de hardware em um computador desmontado ou através de imagens:

  • Placa-mãe (motherboard)
  • Processador (CPU)
  • Memória RAM
  • Disco rígido (HD/SSD)
  • Fonte de alimentação
  • Placa de vídeo (GPU)
  • Slots de expansão
  • Barramentos
✅ Resolução Completa

1. Placa-Mãe (Motherboard)

  • Função: PCB principal que interconecta todos os componentes
  • Componentes integrados: Chipset, BIOS/UEFI, slots, conectores
  • Características: Form factor (ATX, micro-ATX), socket do processador
  • Importância: Define compatibilidade e capacidade de expansão

2. Processador (CPU - Central Processing Unit)

  • Função: Executa instruções e processa dados
  • Componentes: ULA (Unidade Lógica e Aritmética), UC (Unidade de Controle), Registradores, Cache
  • Características: Clock speed (GHz), número de núcleos, arquitetura (x86, ARM)
  • Exemplos: Intel Core i7, AMD Ryzen 7
  • Analogia: É o "cérebro" do computador

3. Memória RAM (Random Access Memory)

  • Função: Armazenamento temporário e rápido de dados em uso
  • Características: Volátil (perde dados sem energia), acesso aleatório
  • Tipos: DDR4, DDR5 (Double Data Rate)
  • Capacidade típica: 8GB a 32GB em PCs modernos
  • Velocidade: Medida em MHz (ex: 3200 MHz)

4. Armazenamento (HD/SSD)

  • HD (Hard Disk):
    • Armazenamento magnético com partes móveis
    • Maior capacidade, menor custo por GB
    • Velocidade: 5400-7200 RPM
    • Mais lento e suscetível a impactos
  • SSD (Solid State Drive):
    • Armazenamento flash (sem partes móveis)
    • Muito mais rápido (até 10x)
    • Mais caro por GB
    • Menor consumo de energia e silencioso
  • Função: Armazenamento permanente (não-volátil) de SO, programas e dados

5. Fonte de Alimentação (PSU - Power Supply Unit)

  • Função: Converte AC (127V/220V) para DC (12V, 5V, 3.3V)
  • Potência: Medida em Watts (400W, 600W, 800W, etc.)
  • Certificação: 80 Plus (Bronze, Silver, Gold, Platinum - eficiência)
  • Importância: Fornece energia estável para todos os componentes

6. Placa de Vídeo (GPU - Graphics Processing Unit)

  • Função: Processamento de gráficos e imagens
  • Tipos:
    • Integrada: Embutida no processador (Intel UHD, AMD Radeon Vega)
    • Dedicada: Placa separada (NVIDIA GeForce, AMD Radeon)
  • Uso: Games, edição de vídeo, machine learning, mineração
  • Memória própria: VRAM (2GB a 24GB)

7. Slots de Expansão

  • PCI Express (PCIe): Placa de vídeo, SSD NVMe, placas de rede
  • M.2: SSDs compactos de alta velocidade
  • SATA: HD/SSD tradicionais
  • Função: Permite adicionar novos componentes

8. Barramentos

  • Função: Vias de comunicação entre componentes
  • Tipos:
    • Barramento de Dados: Transporta informações
    • Barramento de Endereços: Especifica origem/destino
    • Barramento de Controle: Sincroniza operações
  • Largura: 32 bits, 64 bits (mais bits = mais dados simultaneamente)

⚠️ Erros Comuns dos Alunos:
  • Confundir RAM com armazenamento (HD/SSD)
  • Não diferenciar CPU de GPU

💡 Dicas Pedagógicas:

Descrição dos Componentes:

1. CPU (Unidade Central de Processamento)

  • UC (Unidade de Controle): Busca instruções, decodifica e coordena execução
  • ULA (Unidade Lógica e Aritmética): Executa operações matemáticas e lógicas
  • Registradores: Memória ultrarrápida interna da CPU (PC, IR, ACC, etc.)

2. Memória Principal (RAM)

  • Armazena instruções do programa
  • Armazena dados sendo processados
  • Característica von Neumann: programa armazenado (stored program)

3. Barramentos

  • Barramento de Dados: Transporta dados entre componentes (bidirecional)
  • Barramento de Endereços: Especifica localização na memória (unidirecional: CPU → Memória)
  • Barramento de Controle: Sinais de controle (read, write, clock, interrupt, etc.)

4. Dispositivos de E/S (Input/Output)

  • Entrada: Teclado, mouse, sensores, microfone, câmera
  • Saída: Monitor, impressora, alto-falante, atuadores
  • Bidirecionais: Rede, armazenamento (HD/SSD), touchscreen

Ciclo Fetch-Decode-Execute:

  1. Fetch (Busca): UC busca próxima instrução na memória usando PC (Program Counter)
  2. Decode (Decodificação): UC decodifica a instrução para entender o que fazer
  3. Execute (Execução): ULA executa a operação, resultado vai para registrador/memória
  4. PC incrementado: Aponta para próxima instrução
  5. Repete: Ciclo se repete milhões/bilhões de vezes por segundo

📅 Semana 2: Sistemas Numéricos

Exercício 1: Conversões Numéricas

Objetivo: Praticar conversão entre bases numéricas

Duração: 30 minutos | Peso: 10 pontos

📋 Descrição da Atividade

Os alunos devem realizar as seguintes conversões:

  1. Decimal para Binário: 157, 255, 1024
  2. Binário para Decimal: 11010110, 10101010, 11111111
  3. Decimal para Hexadecimal: 255, 4096, 2048
  4. Hexadecimal para Decimal: 0xFF, 0x1A3, 0xDEADBEEF
  5. Binário para Hexadecimal: 11011010, 10110100
  6. Hexadecimal para Binário: 0xA5, 0xF0
✅ Resolução Completa

1. Decimal para Binário

a) 157₁₀ → Binário

Método: Divisões sucessivas por 2
157 ÷ 2 = 78 resto 1  ↑
 78 ÷ 2 = 39 resto 0  │
 39 ÷ 2 = 19 resto 1  │
 19 ÷ 2 =  9 resto 1  │ Lê de baixo para cima
  9 ÷ 2 =  4 resto 1  │
  4 ÷ 2 =  2 resto 0  │
  2 ÷ 2 =  1 resto 0  │
  1 ÷ 2 =  0 resto 1  ↓

Resposta: 157₁₀ = 10011101₂

Verificação:
1×128 + 0×64 + 0×32 + 1×16 + 1×8 + 1×4 + 0×2 + 1×1
= 128 + 16 + 8 + 4 + 1 = 157 ✓

b) 255₁₀ → Binário

Resposta: 255₁₀ = 11111111₂
(Todos os 8 bits ligados = 2⁸ - 1 = 255)

Verificação:
1×128 + 1×64 + 1×32 + 1×16 + 1×8 + 1×4 + 1×2 + 1×1
= 128 + 64 + 32 + 16 + 8 + 4 + 2 + 1 = 255 ✓

c) 1024₁₀ → Binário

1024 = 2¹⁰ (potência perfeita de 2)
Resposta: 1024₁₀ = 10000000000₂ (11 bits: 1 seguido de 10 zeros)

Verificação: 2¹⁰ = 1024 ✓

2. Binário para Decimal

a) 11010110₂ → Decimal

Posições: 7  6  5  4  3  2  1  0
Bits:     1  1  0  1  0  1  1  0
Pesos:   128 64 32 16  8  4  2  1

Cálculo:
1×128 + 1×64 + 0×32 + 1×16 + 0×8 + 1×4 + 1×2 + 0×1
= 128 + 64 + 16 + 4 + 2
= 214

Resposta: 11010110₂ = 214₁₀

b) 10101010₂ → Decimal

1×128 + 0×64 + 1×32 + 0×16 + 1×8 + 0×4 + 1×2 + 0×1
= 128 + 32 + 8 + 2
= 170

Resposta: 10101010₂ = 170₁₀
(Padrão alternado de bits)

c) 11111111₂ → Decimal

Resposta: 11111111₂ = 255₁₀
(Todos os 8 bits ligados = 2⁸ - 1)

3. Decimal para Hexadecimal

a) 255₁₀ → Hexadecimal

Método: Divisões sucessivas por 16
255 ÷ 16 = 15 resto 15  (15 em hex = F)
 15 ÷ 16 =  0 resto 15  (15 em hex = F)

Resposta: 255₁₀ = 0xFF₁₆

Verificação: F×16 + F×1 = 15×16 + 15 = 240 + 15 = 255 ✓

b) 4096₁₀ → Hexadecimal

4096 ÷ 16 = 256 resto  0
 256 ÷ 16 =  16 resto  0
  16 ÷ 16 =   1 resto  0
   1 ÷ 16 =   0 resto  1

Resposta: 4096₁₀ = 0x1000₁₆

Verificação: 1×16³ = 1×4096 = 4096 ✓

c) 2048₁₀ → Hexadecimal

2048 ÷ 16 = 128 resto 0
 128 ÷ 16 =   8 resto 0
   8 ÷ 16 =   0 resto 8

Resposta: 2048₁₀ = 0x800₁₆

Verificação: 8×16² = 8×256 = 2048 ✓

4. Hexadecimal para Decimal

a) 0xFF₁₆ → Decimal

F = 15 em decimal
0xFF = F×16¹ + F×16⁰
     = 15×16 + 15×1
     = 240 + 15
     = 255

Resposta: 0xFF₁₆ = 255₁₀

b) 0x1A3₁₆ → Decimal

1×16² + A×16¹ + 3×16⁰
= 1×256 + 10×16 + 3×1
= 256 + 160 + 3
= 419

Resposta: 0x1A3₁₆ = 419₁₀

c) 0xDEADBEEF₁₆ → Decimal

D=13, E=14, A=10, D=13, B=11, E=14, E=14, F=15

0xDEADBEEF = D×16⁷ + E×16⁶ + A×16⁵ + D×16⁴ + B×16³ + E×16² + E×16¹ + F×16⁰

Cálculo:
13×268435456 + 14×16777216 + 10×1048576 + 13×65536 + 11×4096 + 14×256 + 14×16 + 15
= 3489660928 + 234881024 + 10485760 + 851968 + 45056 + 3584 + 224 + 15
= 3735928559

Resposta: 0xDEADBEEF₁₆ = 3735928559₁₀
(Número mágico famoso em programação para debug)

5. Binário para Hexadecimal

a) 11011010₂ → Hexadecimal

Método: Agrupar de 4 em 4 bits (da direita para esquerda)
1101 1010
 D    A

1101₂ = 13₁₀ = D₁₆
1010₂ = 10₁₀ = A₁₆

Resposta: 11011010₂ = 0xDA₁₆

Verificação: D×16 + A = 13×16 + 10 = 208 + 10 = 218
            11011010₂ = 1×128 + 1×64 + 0×32 + 1×16 + 1×8 + 0×4 + 1×2 + 0×1
                      = 128 + 64 + 16 + 8 + 2 = 218 ✓

b) 10110100₂ → Hexadecimal

1011 0100
 B    4

1011₂ = 11₁₀ = B₁₆
0100₂ =  4₁₀ = 4₁₆

Resposta: 10110100₂ = 0xB4₁₆

Verificação: B×16 + 4 = 11×16 + 4 = 176 + 4 = 180
            10110100₂ = 128 + 32 + 16 + 4 = 180 ✓

6. Hexadecimal para Binário

a) 0xA5₁₆ → Binário

Método: Converter cada dígito hex em 4 bits binários

A₁₆ = 10₁₀ = 1010₂
5₁₆ =  5₁₀ = 0101₂

Resposta: 0xA5₁₆ = 10100101₂

Verificação: 10100101₂ = 128 + 32 + 4 + 1 = 165₁₀
            0xA5₁₆ = 10×16 + 5 = 160 + 5 = 165₁₀ ✓

b) 0xF0₁₆ → Binário

F₁₆ = 15₁₀ = 1111₂
0₁₆ =  0₁₀ = 0000₂

Resposta: 0xF0₁₆ = 11110000₂

Verificação: 11110000₂ = 128 + 64 + 32 + 16 = 240₁₀
            0xF0₁₆ = 15×16 + 0 = 240₁₀ ✓

Exercício 2: Circuito LED Piscante

Objetivo: Montar primeiro circuito com LED e programar blink

Duração: 35 minutos | Peso: 10 pontos

📋 Descrição da Atividade

Os alunos devem:

  1. Montar circuito com LED e resistor no protoboard
  2. Conectar ao pino digital 13 do Arduino
  3. Programar LED para piscar em intervalos de 1 segundo
  4. Modificar para criar padrão personalizado (SOS em Morse)
✅ Resolução Completa

Parte 1: Montagem do Circuito

Diagrama do Circuito:

Arduino Uno                    Protoboard
┌─────────┐
│         │
│  PIN 13 ├──────────────┬───[R 220Ω]───┬───[LED]───┬
│         │              │               │     +     │
│         │              │               └───────────┘
│     GND ├──────────────┴───────────────────────────┘
│         │
└─────────┘

Componentes necessários:
• 1x LED (qualquer cor, preferencialmente vermelho ou verde)
• 1x Resistor 220Ω ou 330Ω (vermelho-vermelho-marrom ou laranja-laranja-marrom)
• 2x Jumpers macho-macho
• 1x Protoboard

Cálculo do Resistor:
Tensão do pino: 5V
Tensão do LED (vermelho): ~2V
Corrente desejada: ~10-20mA

Lei de Ohm: R = V / I
R = (5V - 2V) / 0.02A = 3V / 0.02A = 150Ω

Resistor comercial próximo: 220Ω (valor seguro e comum)

Com 220Ω: I = 3V / 220Ω ≈ 13.6mA ✓ (valor adequado)

Passo a passo da montagem:

  1. Identifique o polo positivo do LED (perna mais longa = ânodo +)
  2. Insira o LED no protoboard
  3. Conecte o resistor em série com o LED
  4. Ligue um jumper do pino 13 ao resistor
  5. Ligue outro jumper do GND ao cátodo do LED
  6. Verifique: pino 13 → resistor → LED(+) → LED(-) → GND

Parte 2: Programação Básica (Blink)

/* * Programa: LED Piscante Básico * Autor: [Nome do Aluno] * Data: [Data] * Descrição: Pisca LED conectado ao pino 13 a cada 1 segundo */ // Definição de constantes const int LED_PIN = 13; // Pino onde LED está conectado const int DELAY_TIME = 1000; // Tempo em milissegundos (1000ms = 1s) // Função setup: executada uma vez no início void setup() { // Configura pino 13 como saída pinMode(LED_PIN, OUTPUT); // Opcional: Inicializa comunicação serial para debug Serial.begin(9600); Serial.println("LED Blink iniciado!"); } // Função loop: executada repetidamente void loop() { // Liga o LED (nível lógico HIGH = 5V) digitalWrite(LED_PIN, HIGH); Serial.println("LED: LIGADO"); // Aguarda 1 segundo delay(DELAY_TIME); // Desliga o LED (nível lógico LOW = 0V) digitalWrite(LED_PIN, LOW); Serial.println("LED: DESLIGADO"); // Aguarda 1 segundo delay(DELAY_TIME); // Loop reinicia automaticamente }

Parte 3: Padrão Personalizado - SOS em Código Morse

/* * Programa: SOS em Código Morse * S = · · · (3 piscadas curtas) * O = ─ ─ ─ (3 piscadas longas) * S = · · · (3 piscadas curtas) */ const int LED_PIN = 13; const int DOT_TIME = 200; // Ponto: 200ms const int DASH_TIME = 600; // Traço: 600ms (3x ponto) const int SYMBOL_GAP = 200; // Pausa entre símbolos const int LETTER_GAP = 600; // Pausa entre letras const int WORD_GAP = 1400; // Pausa entre palavras void setup() { pinMode(LED_PIN, OUTPUT); Serial.begin(9600); Serial.println("SOS em Morse iniciado!"); } // Função para piscar um ponto (curto) void dot() { digitalWrite(LED_PIN, HIGH); delay(DOT_TIME); digitalWrite(LED_PIN, LOW); delay(SYMBOL_GAP); } // Função para piscar um traço (longo) void dash() { digitalWrite(LED_PIN, HIGH); delay(DASH_TIME); digitalWrite(LED_PIN, LOW); delay(SYMBOL_GAP); } // Função para letra S (· · ·) void letterS() { Serial.print("S "); dot(); dot(); dot(); delay(LETTER_GAP); } // Função para letra O (─ ─ ─) void letterO() { Serial.print("O "); dash(); dash(); dash(); delay(LETTER_GAP); } void loop() { Serial.println("Transmitindo SOS..."); // Transmite S letterS(); // Transmite O letterO(); // Transmite S letterS(); // Pausa antes de repetir Serial.println("\n"); delay(WORD_GAP); }

⚠️ Erros Comuns dos Alunos:
  • Polaridade do LED invertida: LED não acende (cátodo no +)
  • Sem resistor: LED queima ou corrente excessiva
  • Resistor muito alto: LED acende muito fraco
  • pinMode não configurado: Pino em alta impedância
  • Delay muito curto: LED pisca rápido demais, invisível
  • Esquecer digitalWrite LOW: LED fica sempre ligado
💡 Dicas Pedagógicas:
  • Mostre LED queimando (propositalmente) sem resistor para impactar
  • Use multímetro para medir corrente real no circuito
  • Explique: digitalWrite HIGH = 5V, LOW = 0V (níveis TTL)
  • Relacione com arquitetura: pino = 1 bit de saída do microcontrolador
  • Desafio extra: implementar todo o alfabeto Morse
  • Conecte ao conceito de I/O de von Neumann visto na Semana 6
🎓 Conceitos de Arquitetura Relacionados
  • Saída Digital: Pino GPIO (General Purpose Input/Output) do ATmega328P
  • Registradores: digitalWrite modifica bit em registrador de porta (PORTB)
  • Clock: delay() conta ciclos de clock (16MHz no Arduino Uno)
  • Programa Armazenado: Código compilado fica na Flash (32KB)
  • Loop Infinito: Arquitetura embarcada roda continuamente (não "termina")
