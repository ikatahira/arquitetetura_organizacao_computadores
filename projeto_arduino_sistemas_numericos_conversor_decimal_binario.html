🔢 Entendendo Números Binários

Antes de montar, vamos entender o que o circuito vai fazer!

🧠 O que é binário?

No sistema decimal usamos 10 dígitos (0 a 9). No sistema binário usamos apenas 2 dígitos: 0 e 1. Um LED apagado representa o bit 0, e um LED aceso representa o bit 1.

🔦 Como funciona com 8 LEDs?

Pino: D9 D8 D7 D6 D5 D4 D3 D2 Bit: 7 6 5 4 3 2 1 0 Peso: 128 64 32 16 8 4 2 1 Exemplo: número 13 = 8+4+1 = 00001101 D9 D8 D7 D6 D5 D4 D3 D2 [0] [0] [0] [0] [1] [1] [0] [1] OFF OFF OFF OFF ON ON OFF ON ← LEDs

    🔧 Montagem — Parte 1: Posicionando LEDs

    Vamos montar os 8 LEDs na protoboard de forma organizada, um ao lado do outro.

    Disposição dos LEDs na Protoboard

    Protoboard (visão simplificada) A B C D E || F G H I J ------------------------------------- Coluna 3 ● (+) || ● (-) Coluna 5 ● (+) || ● (-) Coluna 7 ● (+) || ● (-) Coluna 9 ● (+) || ● (-) Coluna 11 ● (+) || ● (-) Coluna 13 ● (+) || ● (-) Coluna 15 ● (+) || ● (-) Coluna 17 ● (+) || ● (-) ANODO (+) → lado esquerdo (A–E) CATODO (-) → lado direito (F–J) (O LED deve atravessar o sulco central) LED8 LED7 LED6 LED5 LED4 LED3 LED2 LED1 D9 D8 D7 D6 D5 D4 D3 D2 ← Pinos do Arduino
    1. Pegue os 8 LEDs vermelhos
    2. Identifique a perna longa (+) (ânodo) e a perna curta (−) (cátodo) de cada um
    3. Insira o primeiro LED na coluna 3, atravessando o sulco central da protoboard:
      → Perna longa (+) no lado esquerdo (A–E)
      → Perna curta (−) no lado direito (F–J)
    4. Repita para os outros 7 LEDs, utilizando as colunas 5, 7, 9, 11, 13, 15 e 17 (espaçamento de 2 colunas)
    5. Todos os LEDs devem estar atravessando o meio da protoboard, na vertical, sem encostar uns nos outros
    6. Importante: nunca coloque as duas pernas do LED no mesmo lado (A–E ou F–J), pois isso causa curto-circuito
    ⚠️ Atenção à polaridade!
    LED TEM POLARIDADE — não funciona invertido.
    Perna LONGA = Positivo (+) = Anodo
    Perna CURTA = Negativo (−) = Catodo
    💡 Dica: Olhe pela lateral: a perna longa geralmente fica do lado achatado do bulbo.

    📸 Checkpoint

  • 8 LEDs inseridos na protoboard
  • Todos com perna longa na linha A
  • Todos com perna curta na linha B
  • Nenhum LED torto ou tocando outro

    🔧 Montagem — Parte 2: Resistores 330Ω

    Cada LED precisa de um resistor para limitar a corrente e não queimar.

    Identificando o Resistor 330Ω

    Código de cores: 🟠 laranja — 🟠 laranja — 🟤 marrom — (tolerância) Laranja = 3 Laranja = 3 Marrom = ×10 Resultado: 33 × 10 = 330 Ω ✅

    Onde colocar cada resistor

    Cada resistor fica abaixo do LED, na mesma coluna, ligando o cátodo (−) até a parte inferior da protoboard, que depois vai para o GND do Arduino.

    Coluna: 3 5 7 9 11 13 15 17 │ │ │ │ │ │ │ │ LED ● ● ● ● ● ● ● ● (ânodo +) ↑ │ (cátodo −)● │ Resistor [330Ω] [330Ω] [330Ω] [330Ω] [330Ω] [330Ω] [330Ω] [330Ω] │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ Inferior ● ● ● ● ● ● ● ● (−) trilha negativa → Conectar ao GND do Arduino Fios digitais: D9 D8 D7 D6 D5 D4 D3 D2 │ │ │ │ │ │ │ │ (anodo de cada LED)
    1. Insira o LED na coluna desejada, atravessando o sulco central da protoboard (uma perna de cada lado).
    2. Conecte a perna longa (ânodo +) ao pino digital correspondente do Arduino (D9 até D2).
    3. Deixe a perna curta (cátodo −) posicionada na parte inferior da protoboard, no mesmo lado do resistor.
    4. Coloque o resistor de 330Ω na mesma coluna do cátodo, logo abaixo dele.
    5. Conecte a outra perna do resistor à trilha negativa (−) da protoboard.
    6. Ligue a trilha negativa (−) ao pino GND do Arduino.
    💡 Resistor não tem polaridade!
    Pode inserir em qualquer sentido — os dois lados são iguais.

    📸 Checkpoint

  • 8 resistores inseridos, um por LED
  • Uma perna de cada resistor no catodo do LED
  • Outra perna na trilha GND (−) da protoboard
  • Nenhum resistor solto ou caindo

    🔧 Montagem — Parte 3: Jumpers para o Arduino

    Agora conectamos cada LED ao pino digital correspondente do Arduino com jumpers.

    Mapeamento: LED → Pino Arduino

    LED (na protoboard) Pino Arduino Posição (bit) ───────────────────────────────────────────────── DLED1 (direita) → D2 bit 0 (LSB) DLED2 → D3 bit 1 DLED3 → D4 bit 2 DLED4 → D5 bit 3 DLED5 → D6 bit 4 DLED6 → D7 bit 5 DLED7 → D8 bit 6 DLED8 (esquerda) → D9 bit 7 (MSB)
    1. Pegue 8 jumpers
    2. Conecte a linha A (anodo) do DLED1 ao pino D2 do Arduino
    3. Conecte a linha A do DLED2 ao pino D3
    4. Continue até DLED8 → pino D9

    Jumper GND

    1. Conecte um jumper da trilha GND (−) da protoboard
    2. Ao pino GND do Arduino
    🛑 ANTES DE CONTINUAR — Verifique tudo:
    • Cada anodo (+) está conectado ao pino correto?
    • O GND da protoboard está ligado ao GND do Arduino?
    • Nenhum fio está tocando outro por acidente?
    Um erro de fiação pode queimar um LED ou danificar o Arduino!
    💡 Dica de organização:
    Use jumpers de cores diferentes para cada conexão. Facilita muito a identificação de erros!

    📋 Entendendo o Código

    Vamos analisar o código antes de carregar. Entender o código é tão importante quanto montar o circuito!

    📦 Variável Global

    int num = 0; // Armazena o número digitado no Monitor Serial
    ℹ️ Por que global? Porque ela precisa ser acessível tanto no setup() quanto no loop(). Sempre inicializada em 0.

    ⚡ Função setup()

    void setup() { // Define pinos 2 a 9 como SAÍDAS digitais pinMode(2, OUTPUT); pinMode(3, OUTPUT); pinMode(4, OUTPUT); pinMode(5, OUTPUT); pinMode(6, OUTPUT); pinMode(7, OUTPUT); pinMode(8, OUTPUT); pinMode(9, OUTPUT); // Inicia comunicação Serial a 9600 bps Serial.begin(9600); }
    ℹ️ pinMode(pino, OUTPUT) configura o pino como saída — o Arduino vai enviar tensão (5V ou 0V) para ligar/apagar os LEDs.

    🔁 Função loop() — Leitura Serial

    void loop() { if(Serial.available() > 0) { // Lê a string digitada e converte para inteiro num = Serial.readString().toInt(); ... } }
    ℹ️ Como funciona?
    • Serial.available() > 0 → verifica se chegou algo pelo Monitor Serial
    • Serial.readString() → lê o texto digitado (ex: "42")
    • .toInt() → converte "42" (texto) para 42 (número inteiro)

    🔁 Algoritmo de Conversão Decimal → Binário

    int cont = 9; // começa no pino 9 (bit mais significativo) while(num > 0) { if(num % 2 == 0) { digitalWrite(cont, LOW); // bit 0 → LED apagado } else { digitalWrite(cont, HIGH); // bit 1 → LED aceso } delay(20); cont--; // próximo pino (bit menos significativo) num = num / 2; // divide por 2 (extrai próximo bit) } // Apaga os LEDs restantes (bits zero à esquerda) while(cont >= 2) { digitalWrite(cont, LOW); delay(20); cont--; }
    💡 O truque matemático:
    Para extrair os bits de um número, basta dividir por 2 repetidamente e olhar o resto:
    • Resto 0 → bit vale 0 (LED apagado)
    • Resto 1 → bit vale 1 (LED aceso)

    Exemplo: 13 ÷ 2 = 6 resto 1 → 6 ÷ 2 = 3 resto 0 → 3 ÷ 2 = 1 resto 1 → 1 ÷ 2 = 0 resto 1
    Lendo os restos de baixo para cima: 1101 = 13 ✅
  • Entendi o papel da variável global "num"
  • Entendi que setup() configura os pinos como saída
  • Entendi que o loop aguarda dados do Serial Monitor
  • Entendi o algoritmo de divisões por 2

    ⬆️ Carregar o Código no Arduino

    Código Completo — CodigoArduinoIde.ino

    int num = 0; // Variável global para o número recebido void setup() { pinMode(2, OUTPUT); pinMode(3, OUTPUT); pinMode(4, OUTPUT); pinMode(5, OUTPUT); pinMode(6, OUTPUT); pinMode(7, OUTPUT); pinMode(8, OUTPUT); pinMode(9, OUTPUT); Serial.begin(9600); } void loop() { if(Serial.available() > 0) { num = Serial.readString().toInt(); int cont = 9; while(num > 0) { if(num % 2 == 0) { digitalWrite(cont, LOW); delay(20); cont--; } else if(num % 2 == 1) { digitalWrite(cont, HIGH); delay(20); cont--; } num = num / 2; } while(cont >= 2) { digitalWrite(cont, LOW); delay(20); cont--; } } }

    2. Ou pressione Ctrl+Shift+M
    3. Certifique-se de que a velocidade está em 9600 baud (canto inferior direito)

    Testando o Conversor:

    1. No campo de texto, digite 13 e pressione Enter ou clique em "Enviar"
    2. Observe os LEDs na protoboard — eles devem acender no padrão 00001101
    3. LEDs D2, D4 e D5 devem estar acesos (bits 0, 2 e 3)
    4. Tente outros números!
    💡 Números para testar:
    • 1 → apenas D2 aceso (00000001)
    • 255 → todos os 8 LEDs acesos (11111111)
    • 170 → LEDs alternados (10101010)
    • 85 → LEDs alternados invertidos (01010101)
    • 0 → todos apagados (00000000)
    ⚠️ LEDs não acendem?
    • Verifique se a velocidade do Monitor Serial está em 9600 baud
    • Confira as conexões dos jumpers (pino correto por LED)
    • Verifique a polaridade dos LEDs (perna longa no anodo)

    🏆 Desafios

    Agora que tudo funciona, vamos ir além! Tente os desafios abaixo:

    🔴 Desafio 1 — Contador automático:

    Modifique o código para que o Arduino conte de 0 a 255 automaticamente, sem precisar do Monitor Serial:

    void loop() { for(int i = 0; i <= 255; i++) { // mostre o número 'i' nos LEDs // (adapte o algoritmo de conversão aqui!) delay(500); // 0,5 segundo por número } }

    🟡 Desafio 2 — Contagem regressiva:

    Faça o contador ir de 255 até 0 e depois reiniciar:

    for(int i = 255; i >= 0; i--) { // mostre 'i' nos LEDs delay(200); }

    🟢 Desafio 3 — Potência de 2:

    Faça um único LED "andar" da direita para a esquerda (1, 2, 4, 8, 16, 32, 64, 128):

    for(int i = 0; i < 8; i++) { int valor = 1 << i; // 2^i usando deslocamento de bits // mostre 'valor' nos LEDs delay(300); }
    💡 Lembre-se: Após modificar o código, sempre:
    1. Verificar (Ctrl+R)
    2. Carregar (Ctrl+U)
    3. Observar o resultado nos LEDs!
