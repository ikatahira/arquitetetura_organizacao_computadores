Instalação de Sistemas Operacionais
com VirtualBox

Aprenda a criar máquinas virtuais e instalar Ubuntu Desktop, Ubuntu Server e Windows 11 no Oracle VirtualBox — compreendendo na prática como o SO interage com o hardware virtualizado e gerencia recursos do sistema.

🖥️ VirtualBox 7.x 🐧 Ubuntu 24.04 LTS 🪟 Windows 11 ⏱ ~3–4h de lab 🎯 Conceito: Hypervisor Tipo 2
🔷
Por que VirtualBox? — Hypervisor Tipo 2
Virtualização sem hardware dedicado · Ideal para laboratório
CONCEITO
O QUE É?

O VirtualBox é um hypervisor Tipo 2 (hosted) — executa sobre um SO hospedeiro (host) e cria máquinas virtuais (guests) que compartilham o hardware físico. Diferente do Tipo 1 (bare-metal), ele não precisa de hardware dedicado, ideal para laboratórios e estudo.

COMO FUNCIONA?

A CPU do host precisa ter virtualização por hardware (VT-x / AMD-V) habilitada no BIOS/UEFI. O VirtualBox emula dispositivos como HD virtual (VDI), placa de rede e controlador USB, permitindo que o SO guest acredite estar em hardware real.

🏗️ Pilha de Camadas: Hardware Físico → BIOS/UEFI → SO Host (Windows/Linux) → VirtualBox (Hypervisor T2) → Máquina Virtual (SO Guest) → Aplicações do Guest
⚙️ PASSO INICIAL — Verificar Virtualização no BIOS
1
Reiniciar e entrar no BIOS/UEFI
Ao ligar o computador, pressione Del, F2 ou F10 (depende do fabricante) para acessar o BIOS. Procure por Virtualization Technology, VT-x ou AMD-V e habilite se estiver desativado.
2
Instalar o VirtualBox 7.x + Extension Pack
Baixe o instalador em virtualbox.org e instale normalmente. Em seguida, baixe o VirtualBox Extension Pack (mesma página) e instale pelo menu Arquivo → Preferências → Extensões para habilitar suporte USB 3.0 e outros recursos.
💡 O Extension Pack é gratuito para uso pessoal/educacional, mas requer licença comercial para empresas.
3
Baixar as ISOs antes do lab
Use os links da sidebar para baixar: Ubuntu 24.04 Desktop (~5,5 GB), Ubuntu 24.04 Server (~2,5 GB) e o Media Creation Tool do Windows 11 (gera ISO ~6 GB). Isso deve ser feito com antecedência para não comprometer o tempo de lab.
⚠️ Verifique o hash SHA256 das ISOs após o download para garantir integridade do arquivo.

Interface gráfica GNOME · Ideal para desktop e aprendizado
INICIANTE

SOBRE O SO

Ubuntu Desktop é a versão com interface gráfica GNOME, voltada ao uso cotidiano. Possui suporte LTS (Long Term Support) por 5 anos, kernel Linux 6.8+, gerenciador de pacotes APT e repositórios extensos. Ideal para o primeiro contato com Linux em ambiente gráfico.

📋 PASSO A PASSO — CRIAÇÃO E INSTALAÇÃO
1
Criar nova VM no VirtualBox
Clique em Novo no VirtualBox Manager. No campo ISO Image, selecione a ISO do Ubuntu Desktop. O VirtualBox preencherá automaticamente tipo e versão. Marque "Skip Unattended Installation" para instalar manualmente e aprender o processo.
📸 [Captura: tela "Criar máquina virtual" com ISO selecionada]
2
Configurar Hardware da VM
Defina RAM (≥ 2048 MB), número de vCPUs (≥ 2) e habilite a opção EFI (UEFI) na aba Sistema → Placa-mãe se quiser simular boot moderno. Na aba Exibição, aloque 128 MB de vídeo e habilite Aceleração 3D.
💡 O Ubuntu 24.04 suporta UEFI nativamente. Usar EFI é mais realista e prepara para o particionamento com GPT.
3
Criar Disco Virtual (VDI)
Selecione VDI (VirtualBox Disk Image), alocação dinâmica e tamanho de 25 GB. Alocação dinâmica só ocupa o espaço real usado no disco do host, crescendo conforme necessário até o limite.
📸 [Captura: configuração de disco virtual — tipo VDI, alocação dinâmica]
4
Inicializar a VM e selecionar "Try or Install Ubuntu"
Inicie a VM. O GRUB mostrará a tela inicial — selecione "Try or Install Ubuntu". O ambiente Live carregará o Ubuntu sem instalar, permitindo testar hardware e rede. Clique em "Install Ubuntu" na área de trabalho para iniciar a instalação.
5
Configurar idioma, teclado e tipo de instalação
Escolha Português (Brasil) como idioma. Na tela de tipo de instalação, selecione "Instalação Normal" (inclui apps de escritório) ou "Instalação Mínima" para apenas o sistema base. Marque "Baixar atualizações durante a instalação".
6
Particionamento — "Apagar disco e instalar Ubuntu"
Como o disco virtual é exclusivo da VM, selecione "Apagar disco e instalar Ubuntu". O instalador criará automaticamente as partições: /boot/efi (FAT32, ~512 MB), / (ext4, restante) e swap. Clique em Instalar Agora.
⚠️ Esta opção apagaria todo o disco em uma máquina real. Em VM é seguro pois o disco é virtual e isolado.
7
Definir fuso horário, usuário e senha
Selecione São Paulo como fuso horário. Crie um usuário com nome, nome de computador (hostname) e senha. Marque "Exigir senha para entrar". Anote bem as credenciais — serão usadas em todos os labs com esse SO.
8
Aguardar instalação e reiniciar
O processo leva ~15 minutos. Ao concluir, clique em Reiniciar Agora e pressione Enter quando solicitado para ejetar a mídia. O VirtualBox faz isso automaticamente. Faça login com as credenciais criadas.
📸 [Captura: desktop GNOME do Ubuntu 24.04 após primeiro login]
9
Instalar Guest Additions (essencial!)
No menu VirtualBox, clique em Dispositivos → Inserir imagem de CD do Guest Additions. No Ubuntu, abra o terminal e execute:
# Atualizar e instalar dependências
$ sudo apt update && sudo apt install build-essential dkms linux-headers-$(uname -r) -y
# Montar e executar o instalador
$ sudo mount /dev/cdrom /mnt
$ sudo /mnt/VBoxLinuxAdditions.run
💡 Após reiniciar, você terá: resolução de tela automática, área de transferência compartilhada, arrastar & soltar e pasta compartilhada com o host.

Guia de Instalação — Ubuntu Server 24.04 LTS
Sem interface gráfica · CLI puro · Modo servidor de produção
INTERMEDIÁRIO
CONFIG. DA VM
NomeUbuntu-Server-24.04
Tipo / VersãoLinux / Ubuntu 24.04 LTS (64-bit)
RAM1024 MB (mín) — 2048 MB (rec)
CPU (vCores)1 (mín) — 2 (rec)
HDD VirtualVDI · Alocar dinâmico · 20 GB
RedeNAT + Host-Only Adapter
ExibiçãoSem aceleração 3D (não há GUI)
SOBRE O SO

Ubuntu Server opera sem interface gráfica — tudo via terminal (CLI). Consome muito menos RAM e CPU que o Desktop. É a base de infraestrutura de servidores web, bancos de dados e containers Docker. O processo de instalação usa um instalador textual interativo (Subiquity).

📋 PASSO A PASSO — INSTALAÇÃO
1
Criar a VM — Configurações para servidor
Crie a VM com a ISO do Ubuntu Server. Não habilite aceleração 3D nem áudio. Configure dois adaptadores de rede: Adaptador 1 = NAT (acesso à internet), Adaptador 2 = Host-Only Adapter (para acessar via SSH do host). Isso simula um servidor real em rede.
2
Boot e seleção de idioma no instalador textual
O instalador Subiquity é totalmente textual — use ↑↓ para navegar e Enter para confirmar. Selecione English (recomendado para servidores — evita caracteres especiais nos logs) e layout de teclado Português Brasil.
📸 [Captura: tela inicial do instalador Subiquity (textual)]
3
Tipo de instalação — Ubuntu Server
Escolha "Ubuntu Server" (não "Ubuntu Server (minimized)"). A versão minimizada remove utilitários úteis para aprendizado. Confirme a detecção da interface de rede — o Subiquity configurará o DHCP automaticamente via NAT.
4
Configuração de disco — LVM
Selecione "Use an entire disk" e marque "Setup this disk as an LVM group". O LVM (Logical Volume Manager) permite redimensionar volumes em produção sem reformatar. O instalador criará: partição EFI, /boot e um volume lógico raiz (/) no grupo VG.
💡 LVM é padrão em servidores Ubuntu — o aluno verá depois como expandir o volume com lvextend e resize2fs.
5
Criar perfil — usuário, hostname e senha
Defina: seu nome completo, nome do servidor (ex: server-arq001), nome de usuário (ex: aluno) e senha. Em servidores, o hostname é importante pois aparece nos logs e é usado pelo SSH e serviços de rede.
6
Habilitar OpenSSH Server (OBRIGATÓRIO)
Na tela de pacotes opcionais, marque "Install OpenSSH server". Isso instala e habilita o serviço SSH automaticamente, permitindo acesso remoto à VM sem precisar usar o terminal do VirtualBox — simula acesso a servidor real.
⚠️ Sem o SSH habilitado, você precisará usar o console da VM — muito mais limitado. SSH é essencial para servidores.
7
Aguardar instalação e reiniciar
A instalação do servidor leva ~10 minutos (sem GUI). Ao finalizar, selecione "Reboot Now". O sistema inicializará em modo texto. Faça login com seu usuário e senha.
📸 [Captura: prompt de login do Ubuntu Server após boot]
8
Primeiros comandos pós-instalação
Após o login, execute para confirmar que tudo está funcionando:
# Atualizar o sistema
$ sudo apt update && sudo apt upgrade -y

# Verificar IP da interface Host-Only (para SSH)
$ ip addr show

# Verificar status do SSH
$ sudo systemctl status ssh

# No host: conectar via SSH (substituir <IP>)
$ ssh aluno@<IP-da-VM>
💡 Uma vez que o SSH estiver funcionando, você pode fechar a janela do VirtualBox e trabalhar 100% pelo terminal do seu SO host — exatamente como em servidores de produção.

Guia de Instalação — Windows 11 no VirtualBox
TPM 2.0 virtual · Secure Boot · Requisitos especiais de hardware
AVANÇADO
⚠️ Atenção: O Windows 11 exige TPM 2.0 e Secure Boot — recursos que o VirtualBox emula a partir da versão 6.1.34+. A configuração é mais complexa que no Linux. Siga os passos cuidadosamente.
CONFIG. DA VM
NomeWindows11-VM
Tipo / VersãoMicrosoft Windows / Win 11 (64-bit)
RAM4096 MB (mín) — 8192 MB (rec)
CPU (vCores)2 (mín) — 4 (rec)
HDD VirtualVDI · Alocar dinâmico · 64 GB
RedeNAT
FIRMWAREEFI (Secure Boot habilitado)
POR QUE É MAIS COMPLEXO?

O Windows 11 verifica na instalação: TPM 2.0 (chip de segurança), Secure Boot (UEFI), CPU compatível e ≥4 GB RAM. O VirtualBox emula TPM virtualmente e suporta Secure Boot via EFI, mas a configuração requer passos adicionais não necessários no Linux.

📋 PASSO A PASSO — INSTALAÇÃO
1
Baixar a ISO do Windows 11 (Media Creation Tool)
Acesse microsoft.com/pt-br/software-download/windows11 e baixe o Media Creation Tool. Execute-o, escolha "Criar mídia de instalação" e depois "Arquivo ISO". Salve em local acessível (~6 GB). Você precisará de uma licença válida do Windows 11 ou pode usar em modo avaliação por 30 dias.
2
Criar VM com as configurações corretas
Ao criar a VM com a ISO do Win11, o VirtualBox detectará automaticamente e configurará EFI + TPM 2.0 virtual. Confirme que EFI está habilitado em Sistema → Placa-mãe e que o chip de segurança (TPM) está como TPM 2.0 em Sistema → Placa-mãe.
📸 [Captura: configurações de Sistema → Placa-mãe com TPM 2.0 e EFI ativados]
3
Habilitar Nested Paging e PAE/NX
Em Sistema → Processador, habilite PAE/NX. Em Sistema → Aceleração, habilite Nested Paging e VT-x/AMD-V. Essas opções são necessárias para melhor desempenho e compatibilidade com o Hyper-V do Windows.
4
Boot e iniciar instalação
Inicie a VM. Pressione qualquer tecla quando aparecer "Press any key to boot from CD/DVD". O instalador do Windows carregará. Selecione idioma Português (Brasil), formato de hora e teclado ABNT2. Clique em Instalar agora.
5
Chave do produto — pular ou inserir
Se não tiver licença, clique em "Não tenho uma chave de produto". Selecione a edição Windows 11 Pro (mais recursos para lab). Aceite os termos de licença.
💡 Em ambiente educacional, o Windows 11 pode ser usado em modo avaliação por 30 dias sem ativação. Extensões adicionais são possíveis.
6
Tipo de instalação — Personalizada
Selecione "Personalizada: instalar somente o Windows". Na tela de partições, o disco virtual aparece como não alocado. Clique em Novo → o instalador criará automaticamente partições: EFI, MSR (reservada) e a partição principal do Windows (NTFS).
📸 [Captura: tela de particionamento com disco não alocado e partições criadas automaticamente]
7
Aguardar instalação (~30 min) e configuração OOBE
A instalação reiniciará algumas vezes. Após o boot final, o OOBE (Out-of-Box Experience) iniciará: selecione país, teclado ABNT2, conecte à rede (via NAT). Dica importante: se quiser criar conta local (sem Microsoft Account), clique "Opções de entrada" → "Conta offline" → "Experiência limitada".
8
Instalar Guest Additions para Windows
No menu VirtualBox, clique em Dispositivos → Inserir imagem de CD do Guest Additions. No Explorer da VM, abra o CD e execute VBoxWindowsAdditions-amd64.exe. Aceite tudo e reinicie. Após isso, a resolução se ajustará automaticamente e a integração de mouse melhorará.
:: No CMD como Administrador — verificar drivers instalados
C:\> dxdiag
C:\> devmgmt.msc
9
Criar snapshot — Salvar estado limpo da VM
Com o sistema funcionando, crie um snapshot em Máquina → Tirar instantâneo. Nomeie como "Instalação Limpa". Snapshots permitem reverter a VM a qualquer momento — essencial para laboratórios onde erros de configuração podem ocorrer.
💡 Snapshots são como pontos de restauração do sistema, mas no nível da máquina virtual inteira — incluindo RAM, disco e estado da CPU.

🔍 CONCEITOS-CHAVE OBSERVADOS NOS 3 SOs
Kernel e Espaço de Usuário: Todos os 3 separam modo kernel (privilegiado, acessa hardware) do modo usuário (aplicações). O Linux permite ver isso via /proc e /sys.
Gerenciamento de Processos: Linux usa ps aux e top; Windows usa Task Manager. Ambos mostram PID, estado, CPU e memória de cada processo.
Sistema de Arquivos: Linux usa ext4/LVM; Windows usa NTFS. Ambos têm journaling para recuperação de falhas — observável nos logs do SO.
Syscalls: Aplicações usam chamadas de sistema para acessar o kernel — rede, disco, memória. No Linux: strace ls mostra as syscalls em tempo real.

📌 Conexão com a Ementa da Disciplina
🔗 Sistemas Operacionais — Conceitos e Funções: A instalação prática materializa abstrações como kernel, modo usuário/kernel, chamadas de sistema, escalonamento e gerenciamento de memória. O aluno observa o SO real, não apenas diagramas.
🏗️ Hierarquia de Hardware: A virtualização expõe toda a pilha: firmware BIOS/UEFI → bootloader (GRUB/EFI) → kernel → sistema de arquivos → processos. O VirtualBox torna essa hierarquia visível e manipulável.
💡 Lab 12 — FreeRTOS: Após instalar os SOs, o aluno terá referência concreta para comparar o FreeRTOS (RTOS embarcado do ESP32) com SOs de propósito geral — escalonamento, gerenciamento de heap e tasks vs processos.
